/goal 현재 목표는 GitHub main의 실제 상태와 로컬 문서/코드 상태를 재정렬하고, 앞으로의 DCGO headless RL.Engine 이식을 원본 DCGO 구조에 더 가깝게 유지하는 것이다. 전체 목표는 신규 엔진 설계가 아니라 ./DCGO Unity battle 로직의 headless 이식이다. Codex는 먼저 GitHub/로컬 최신 상태를 비교하고, 문서에 남은 과거 ST1 실패 상태와 현재 ST1/ST2/ST3 구현 상태의 모순을 정리해야 한다. 카드 효과는 원본 DCGO/Assets/Scripts/CardEffect/{Set}/{Color}/{CardId}.cs 구조에 대응되는 파일로 유지해야 하며, Catalog는 registry 역할만 담당해야 한다. 공통 service는 원본 AutoProcessing, CardEffectCommons, CardController, AttackProcess, TurnStateMachine 같은 공통 처리에 대응되는 경우에만 유지한다. ST1~ST3 구현을 계속 확장하기 전에 source-aligned 구조 감사, stale 문서 정리, CardEffect 파일 구조 guard, completion gate 정합성 검사를 먼저 수행한다. RL 학습용 ObservationEncoder, RewardCalculator, DatasetExporter, Trainer, RL Environment API는 아직 구현하지 않는다. 기존 DCGO Unity 원본 파일은 수정하지 않는다. remote 없는 로컬 Git만 사용하고 push/fetch/pull 하지 않는다. 각 작업 시작 전후 git status --short, git diff --stat, git diff --name-only를 보고하고, 사용자 승인 없는 commit은 만들지 않는다. 모든 분석, 진행상황, 테스트 결과, TODO는 한국어로 작성한다.
