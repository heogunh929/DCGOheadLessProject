/goal 현재 목표는 DCGO Unity battle 로직을 headless RL.Engine으로 최대한 같은 의미와 구조로 이식하는 것이다. 현재 ST1~ST3 카드별 파일/registry/status validation은 상당히 진행됐지만, 이것을 전체 엔진 완성으로 해석하지 않는다. 다음 작업은 원본 asset CardEffectClassName mapping, option Executing lifecycle, trigger pipeline 필수 구성, security timing, MultipleSkills/AfterEffectsActivate priority, counter/block/attack-target timing, golden scenario, Unity/RL trace parity, ST1~ST3 scope whole-engine completion gate를 순서대로 보강하는 것이다. 카드별 effect body는 원본 CardEffect/{Set}/{Color}/{CardId}.cs에 대응되는 파일에 유지하고 Catalog는 registry만 담당한다. 공통 service는 원본 AutoProcessing, MultipleSkills, CardEffectCommons, CardController, AttackProcess, TurnStateMachine의 공통 책임에 대응할 때만 둔다. 특정 카드 ID shortcut, 직접 zone list 수정, silent no-op, 원본 의미를 깨는 단순화는 금지한다. DCGO/Assets/Scripts 원본은 읽기 전용 Source of Truth로 유지한다. remote가 존재해도 fetch/pull/push는 사용자 명시 승인 없이 실행하지 않는다. commit도 사용자 승인 없이 만들지 않는다. 학습용 ObservationEncoder, RewardCalculator, DatasetExporter, Trainer, RL Environment API는 whole-engine gate 통과와 사용자 승인 전까지 구현하지 않는다. 모든 분석, 진행상황, 테스트 결과, TODO는 한국어로 작성하며 한 번에 queue 항목 하나만 수행한다.
