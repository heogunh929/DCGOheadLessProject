/goal 현재 목표는 GitHub main의 실제 headlessDCGO 진행 상태를 기준으로, DCGO Unity 원본 구조에 최대한 가깝게 RL.Engine 이식 구조를 재정렬하는 것이다. ST2/ST3 추가 구현을 바로 진행하지 말고, 먼저 GitHub main과 로컬 상태를 비교하고, CardEffect가 원본 DCGO/Assets/Scripts/CardEffect/{Set}/{Color}/{CardId}.cs에 대응되는 카드별 파일 구조를 유지하는지 감사한다. Catalog는 registry 등록만 담당해야 하며, 카드별 effect body를 Catalog나 거대한 통합 파일에 몰아넣는 방식은 금지한다. 공통 service는 원본 AutoProcessing, CardEffectCommons, CardController, AttackProcess, TurnStateMachine 같은 공통 처리 로직에 대응될 때만 유지하고, 카드별 구현을 숨기는 통폐합을 금지한다. 이미 구현된 ST1~ST3 코드가 있다면 그 기능을 되돌리지 말고, 원본 구조와의 mapping, 파일 배치, 테스트 guard, 문서 정합성을 먼저 검증/정렬한다. 기존 DCGO Unity 원본 파일은 수정하지 않는다. RL 학습용 ObservationEncoder, RewardCalculator, DatasetExporter, Trainer, RL Environment API는 아직 구현하지 않는다. remote 없는 로컬 Git만 사용하고 push/fetch/pull 하지 않는다. 작업 시작 전후 git status --short, git diff --stat, git diff --name-only를 보고하고, 사용자 승인 없는 commit은 만들지 않는다. 모든 분석, 진행상황, 테스트 결과, TODO는 한국어로 작성한다.
